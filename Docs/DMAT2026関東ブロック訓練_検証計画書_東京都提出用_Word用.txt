1. 目的

本検証は、DMAT2026関東ブロック訓練において、災害対応本部の物資支援に関する情報収集・一元化・意思決定支援の新しい運用を検証することを目的とする。

これまで各本部ごとに手入力のExcelをメール等で共有し、人的リソースを要していた「複数リソースの情報を一つにまとめる」作業を、音声入力と検索AIを軸にリアルタイムで収集・一元化し、欲しい情報を欲しいタイミングで得られる土台を構築する実験である。

具体的には、人の手で集めていた情報をリアルタイムで収集し、収集したものをすぐ意思決定に反映するための基盤をつくることを検証する。


────────────────────────────────────────────────────────────

2. 検証内容

2.1 検証のねらい

項目	内容
情報の一元化	各本部別の要求・報告・指示等を含む情報を、音声のみで収集し、一元的に記録・蓄積する
本部別記録の維持	音声でコミュニケーションをとった際、本部別に記録が残ることを確認する
災害別データの集約	各災害（訓練）のデータが一つのデータベースに収まり、検証・抽出が容易であることを確認する
リアルタイム把握	全体情報をリアルタイムで把握し、意思決定に活用できる状態をつくる
運用負荷の軽減	Excelの手入力・メール共有に依存していた従来運用と比較し、音声だけで収集・まとめることの実現可能性を検証する


2.2 検証のポイント

（1）音声入力による本部別情報の収集
　　物資支援班として、各本部が音声で発話した内容（要求・報告・確認等）が、本部名と紐づいて記録されるか。

（2）一元化された情報の検索・抽出
　　蓄積されたクロノロジー（時系列記録）を、災害・本部・種別（指示／報告／依頼等）でフィルタし、必要な情報をすぐ取り出せるか。

（3）リアルタイムでの全体把握
　　複数本部の情報が、リアルタイムで一元的に参照・集約され、意思決定の材料として利用できるか。

（4）「大きなまとめ」の実現
　　災害現場において、複数リソースの情報を一つにまとめるという、従来きわめて手間のかかる作業を、音声入力と検索AIをきっかけに、大きく簡素化・効率化できるか。


────────────────────────────────────────────────────────────

3. 運用イメージ

3.1 利用者・利用シーン

・利用者：物資支援班に参加する各本部の担当者（都道府県調整本部等）

・利用シーン：訓練時に、Zoom等で音声コミュニケーションを取りながら、物資支援専用UI（上記ユーザー用URL）にアクセスし、本部名を選択して入室。マイクで発話した内容が、その本部として記録される。


3.2 運用フロー（概要）

1. 各本部担当者が「物資支援班」専用URL（ユーザー用）にアクセス
2. 所属本部を選択し、入室
3. Zoom等で音声通話しながら、必要に応じて当システム上で音声入力（Push-to-Talk等）
4. 発話は自動で文字起こし・分類・要約され、本部別にクロノロジーへ記録
5. 関係者は、画面上でリアルタイムに時系列・本部別の記録を確認
6. 検索・フィルタにより、必要な情報を抽出し、意思決定に活用


3.3 データの流れ

・入力：各本部からの音声（および必要に応じたメタ情報）
・処理：音声→文字起こし（STT）→種別分類・要約（LLM）→クロノロジー行として保存
・蓄積：災害（訓練）単位でセッションが紐づき、一つのデータベース（ストレージ）に収まる
・出力：時系列一覧、本部別・種別フィルタ、検索AIを利用した問い合わせ応答


────────────────────────────────────────────────────────────

4. システム概要（検証で使用する構成）

4.1 物資支援ブランチ（専用UI）

・対象災害：「2026年DMAT関東ブロック訓練_物資支援」
・利用URL：上記ユーザー用URLを物資支援班参加者に案内して利用させる。検証側の管理・確認は管理人用URLで行う。
・機能概要：
　　－ 本部選択→入室
　　－ 音声入力（Push-to-Talk等）による発話の送信
　　－ 本部別・時系列のクロノロジー表示
　　－ 種別（指示／依頼／報告／決定／確認／リスク／その他）でのフィルタ
　　－ 検索AI（クロノロジーに対する質問応答）による情報抽出


4.2 本部別記録・災害別データ

・発話は参加者（本部）ごとに紐づいて記録される。
・一つの災害（訓練）に物資支援セッションが紐づき、そのセッション配下に全本部のクロノロジーが蓄積される。
・結果として、災害別にリアル情報が一つのデータベースに収まり、検証・抽出が容易になる。


4.3 リアルタイム性

・音声送信→文字起こし→分類・要約→記録→表示までをパイプライン化し、リアルタイムに近い形で全体情報を把握できるようにする。
・WebSocket等により、新規記録の追加を即時に反映。
